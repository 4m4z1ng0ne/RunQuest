## Актуальный план разработки фронтенда (на текущий момент)

### Реализовано:

- Навигация между экранами (BottomNavigationBar, main_tabs_screen).
- Экраны входа, регистрации, welcome, восстановление сессии.
- Экран профиля: просмотр, редактирование, аватар, уровень, опыт, приватность, просмотр других пользователей.
- Друзья: отправка/принятие/отклонение запросов, список друзей, статус дружбы, поиск пользователей.
- Гонки: карта гонки, старт/финиш, проверка расстояния, результаты, публикация, просмотр маршрута, лидерборд.
- Тренировки: запись, публикация, история активности, результаты, маршрут, фильтрация по типу.
- Достижения и челленджи: экраны, прогресс, присоединение, просмотр чужих достижений/челленджей.
- Совместные пробежки: приглашения, статусы, планирование, добавление участников, старт, отмена, вкладка запланированных пробежек.
- Лента активности: просмотр, переход к деталям, ActivityFeedService.
- Блокировки: блокировка/разблокировка пользователей, отображение статуса.
- Настройки: экран настроек, экран уведомлений (NotificationSettingsScreen), экран выбора единиц измерения, цветовой схемы, политики и т.д.
- UI/UX: обработка ошибок, скроллы, адаптация под разные экраны, базовые анимации.
- Работа с API: через AuthService, FriendService, ActivityFeedService, поддержка refresh токенов, обработка ошибок, автоматическое обновление профиля.
- Модели: поддержка всех ключевых сущностей (User, Run, Challenge, Achievement, JointRunInvitation, PlannedJointRun и др.).

### Частично реализовано/требует доработки:

- Уведомления: реализован только UI для настроек уведомлений (NotificationSettingsScreen), реальной логики push-уведомлений нет (нет интеграции с Firebase/OneSignal и регистрации DeviceToken).
- Расширенная аналитика: есть экран статистики, но нет графиков, расширенных фильтров, визуализации темпа/скорости.
- UI/UX: базовые анимации и переходы есть, но нет сложных анимаций, не реализована оптимизация производительности (например, пропущенные кадры).
- Приватность: реализована базово, но нет экрана управления приватностью, нет полной поддержки приватного профиля и видимости активности.
- Локализация: нет поддержки смены языка/локализации (весь интерфейс на русском).
- Настройки: нет экрана управления push-уведомлениями на уровне системы, нет управления DeviceToken.
- Тесты: нет unit, интеграционных и UI-тестов.

### Не реализовано:

- Push-уведомления: нет интеграции с сервисами push-уведомлений, нет регистрации токенов устройства, нет логики получения/отображения уведомлений.
- Внутриприложные уведомления: нет отображения уведомлений внутри приложения (только UI-переключатели).
- Голосовые уведомления: отсутствуют.
- Форма запроса на создание гонки: отсутствует.
- Полная персонализация: нет установки беговых целей, нет расширенных настроек приватности, нет выбора языка.
- Документирование кода и тесты: отсутствуют.

### Текущий статус соответствия дипломной работе:

Фронтенд реализует все основные пользовательские сценарии, описанные в дипломе: аутентификация, профиль, друзья, гонки, тренировки, достижения, челленджи, совместные пробежки, лента активности, блокировки, базовые настройки. Реализована интеграция с API, поддержка refresh токенов, обработка ошибок, базовая приватность.

Однако, для полного соответствия и повышения качества проекта требуется:
- Реализовать push-уведомления (интеграция, регистрация токенов, получение и отображение).
- Доработать аналитику и статистику (графики, фильтры, визуализация темпа/скорости).
- Улучшить UI/UX (анимации, производительность, соответствие макетам).
- Реализовать расширенные настройки приватности, персонализацию, локализацию.
- Добавить тесты и документацию.
- (Опционально) реализовать форму запроса на создание гонки и голосовые уведомления.
